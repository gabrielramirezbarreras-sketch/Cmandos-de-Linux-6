Practica 1 - Cifrado.
Instalar la herramienta GPG2:
sudo apt update
sudo apt install gnupg2 -y
gpg2 --version

Crear un directorio:
mkdir ~/Desktop/practica_cifrado
cd ~/Desktop/practica_cifrado

Crear un archivo dentro del directorio:
nano mensaje.txt
Nombre = Gabriel Ramirez
Matricula = 2025-1352
Materia = SO3

Verificamos el archivo:
cat mensaje.txt

Cifrar el archivo con GPG2:
gpg2 -c mensaje.txt
ls

Intentar acceder al archivo cifrado:
cat mensaje.txt.gpg

Eliminar el archivo original:
rm mensaje.txt

Descifrar el archivo:
gpg2 mensaje.txt.gpg

Mostrar el contenido del archivo descifrado:
cat mensaje.txt

Practica 2 - IP tables - UFW/Firewall-cmd.
Instalar los servicios HTTP, FTP y SSH:
sudo apt update
sudo apt install apache2 vsftpd openssh-server -y

Iniciar los servicios:
sudo systemctl start apache2
sudo systemctl start vsftpd
sudo systemctl start ssh

Habilitarlos:
sudo systemctl enable apache2
sudo systemctl enable vsftpd
sudo systemctl enable ssh

Verificar que los servicios están funcionando:
sudo systemctl status apache2
sudo systemctl status vsftpd
sudo systemctl status ssh

Verificar los puertos abiertos:
sudo ss -tulnp

Probar los servicios:
Probar HTTP:
http://192.168.140.129

Probar FTP:
ftp 192.168.140.129

Probar SSH:
ssh oem@192.168.140.129

Bloquear los puertos usando IPTABLES:
sudo iptables -A INPUT -p tcp --dport 80 -j DROP
sudo iptables -A INPUT -p tcp --dport 21 -j DROP
sudo iptables -A INPUT -p tcp --dport 22 -j DROP

Ver reglas:
sudo iptables -L

Probar otra vez:
http://192.168.140.129
ssh oem@192.168.140.129
ftp 192.168.140.129

Habilitar nuevamente los puertos con IPTABLES:
sudo iptables -D INPUT -p tcp --dport 80 -j DROP
sudo iptables -D INPUT -p tcp --dport 21 -j DROP
sudo iptables -D INPUT -p tcp --dport 22 -j DROP

Verificamos:
sudo iptables -L

Probar otra vez:
http://192.168.140.129
ftp 192.168.140.129
ssh oem@192.168.140.129

Bloquear puertos usando UFW:
habilitamos UFW.
sudo ufw enable

Ver estado:
sudo ufw status

Bloquear puertos:
sudo ufw deny 80
sudo ufw deny 21
sudo ufw deny 22

Ver reglas:
sudo ufw status numbered

Probar otra vez:
http://192.168.140.129
ftp 192.168.140.129
ssh oem@192.168.140.129

Permitir nuevamente los puertos:
sudo ufw allow 80
sudo ufw allow 21
sudo ufw allow 22

Ver estado:
sudo ufw status

Practica 3 - Instalacion de IDS snort.
Instalar Snort:
sudo apt update
sudo apt install snort -y

Verificar que Snort está instalado:
snort -V

Editar el archivo de reglas de Snort:
sudo nano /etc/snort/rules/local.rules

Crear las reglas de detección:
alert icmp any any -> $HOME_NET any (msg:"ICMP Ping Detectado"; sid:1000001; rev:1;)

alert tcp any any -> $HOME_NET 21 (msg:"Acceso FTP Detectado"; sid:1000002; rev:1;)

alert tcp any any -> $HOME_NET 22 (msg:"Acceso SSH Detectado"; sid:1000003; rev:1;)

alert tcp any any -> $HOME_NET 80 (msg:"Acceso HTTP Detectado"; sid:1000004; rev:1;)

Verificar que Snort cargue las reglas:
sudo snort -T -c /etc/snort/snort.conf

Ejecutar Snort en modo detección:
sudo snort -A console -q -c /etc/snort/snort.conf -i ens33

Probar:
http://192.168.140.129
ftp 192.168.140.129
ssh oem@192.168.140.129
ping 192.168.140.129

Practica 4 - Configurar 2FA con google authenticator Modulo PAM para Acceso SSH.
Instalar Google Authenticator.
sudo apt update
sudo apt install libpam-google-authenticator -y

Configurar Google Authenticator para tu usuario.
google-authenticator

Do you want me to update your "/home/oem/.google_authenticator" file? (y/n) y

Do you want to disallow multiple uses of the same authentication
token? This restricts you to one login about every 30s, but it increases
your chances to notice or even prevent man-in-the-middle attacks (y/n) y

By default, a new token is generated every 30 seconds by the mobile app.
In order to compensate for possible time-skew between the client and the server,
we allow an extra token before and after the current time. This allows for a
time skew of up to 30 seconds between authentication server and client. If you
experience problems with poor time synchronization, you can increase the window
from its default size of 3 permitted codes (one previous code, the current
code, the next code) to 17 permitted codes (the 8 previous codes, the current
code, and the 8 next codes). This will permit for a time skew of up to 4 minutes
between client and server.
Do you want to do so? (y/n) n

If the computer that you are logging into isn't hardened against brute-force
login attempts, you can enable rate-limiting for the authentication module.
By default, this limits attackers to no more than 3 login attempts every 30s.
Do you want to enable rate-limiting? (y/n) y

Se creará el archivo:
~/.google_authenticator

Verificar:
cat ~/.google_authenticator

Configurar PAM para usar Google Authenticator.
Edita el archivo PAM de SSH:
sudo nano /etc/pam.d/sshd

Agrega esta línea al final del archivo.
auth required pam_google_authenticator.so

Configurar SSH para pedir 2FA.
sudo nano /etc/ssh/sshd_config
UsePAM yes
ChallengeResponseAuthentication yes
PasswordAuthentication yes
bdInteractiveAuthentication yes

Reiniciar el servicio SSH.
sudo systemctl restart ssh

Verifica que esté activo:
sudo systemctl status ssh

Desde tu máquina host ejecuta:
